Comentario Pruebas 


Formato


Los archivos de prueba son archivos de texto, pero el formato .trk
Dicho formato, se basa en una línea de caracteres donde la información debe seguir el siguiente orden:

Primero se deberá poner la dimensión del edificio (al ser un cubo, es necesario que sea un número natural), luego un punto y coma (;), y luego una lista de tuplas de 2 elementos separados por una coma (,). Cada una de estas tuplas, deberá contener 2 números naturales, indicando el número de Método (véase Tabla de Resolucion), y el número de Problema (véase Tabla de Problemas). Si se ponen valores que no estén en las tablas, se utilizará un valor por defecto. Las tuplas, están separadas entre sí mediante puntos y coma (;). 

Dadas las dimensiones del edificio, se ha calculado que el número de puertas (cada tupla se asocia a una puerta) del edificio se puede calcular mediante la fórmula:

			F(n)=n**2 * (n-1) + 2(n-1) + F(n-1)

Donde n representa la dimensión del edificio. 
En caso de no introducir suficientes datos no se garantiza que el programa acabe su ejecución debido a falta de datos para poder generar todas las puertas del edificio.


De ésta manera, el programa irá cogiendo datos de las puertas que necesite ir creando. se crean las puertas de una habitación cada vez que se entra en ella (mirando que las puertas no coincidan con las de otras habitaciones). De manera que las 6 primeras puertas se construiran con la primera habitación. Es importante remarcar que el orden que se da en el archivo, no es el orden de las puertas que vamos a atravesar, son los datos que se necesitan para el problema.
